## üéì CORE CONCEPT

Before starting any new project, the AI should:
1. **Start as a college freshman** with basic knowledge
2. **Progress through a complete Ivy League education** in the relevant domain
3. **Achieve post-graduate level expertise** before beginning work
4. **Validate all learning materials** for quality and accuracy

## üìö EDUCATIONAL CURRICULUM STRUCTURE

### Phase 1: Undergraduate Foundation (Freshman ‚Üí Senior)

#### Year 1-2: Foundational Knowledge
- **University Courses to Complete:**
  - MIT OpenCourseWare relevant courses
  - Stanford Online relevant courses
  - Harvard Extension School materials
  - Yale Open Courses in the domain

- **Required Learning:**
  - Core concepts and terminology
  - Historical context and evolution
  - Fundamental theories and principles

#### Year 3-4: Specialized Knowledge
- **University Courses to Complete:**
  - Advanced domain-specific courses
  - Cross-disciplinary applications
  - Case studies and practical applications

- **Required Learning:**
  - Advanced techniques and frameworks
  - Real-world application scenarios
  - Industry tools and technologies
  - Professional standards

### Phase 2: Graduate Education (Master's Level)

#### Master's Program Focus:
- **Research Papers to Review:**
  - Top 20 most-cited papers in the domain (last 10 years)
  - Seminal works that defined the field
  - Recent breakthrough studies
  - Systematic reviews and meta-analyses

- **Advanced Coursework:**
  - Specialized graduate seminars
  - Research methodology
  - Advanced statistical methods
  - Domain-specific advanced techniques

- **Required Learning:**
  - Cutting-edge developments
  - Research gaps and opportunities
  - Controversial topics and debates
  - Emerging trends

### Phase 3: Doctoral-Level Expertise

#### PhD-Level Preparation:
- **Academic Research:**
  - Read 50+ peer-reviewed papers in domain
  - Analyze methodology of top research
  - Understand current research frontiers
  - Identify unanswered questions

- **Expert-Level Knowledge:**
  - Deep technical understanding
  - Ability to critique existing work
  - Recognition of best practices vs. common mistakes
  - Understanding of ethical considerations

## üîç DATA VALIDATION PROTOCOL

### Source Quality Validation

#### Tier 1: Highest Quality (Use First)
- **Academic Sources:**
  - Peer-reviewed journals (Nature, Science, Cell, etc.)
  - University course materials from top 20 institutions
  - Textbooks from recognized authorities
  - Conference proceedings from major conferences

- **Validation Criteria:**
  - ‚úÖ Peer-reviewed publication
  - ‚úÖ Author credentials (PhD, professorship, h-index)
  - ‚úÖ Citation count (>100 for recent papers)
  - ‚úÖ Published by reputable institution
  - ‚úÖ Recent publication (<5 years preferred)

#### Tier 2: High Quality (Use with Verification)
- **Industry Sources:**
  - White papers from established companies
  - Technical documentation from major vendors
  - Industry standards bodies (IEEE, ISO, etc.)
  - Professional organization publications

- **Validation Criteria:**
  - ‚úÖ Author expertise verifiable
  - ‚úÖ Multiple corroborating sources
  - ‚úÖ Technical accuracy can be verified
  - ‚úÖ No obvious commercial bias

#### Tier 3: Supplementary (Use Carefully)
- **Community Sources:**
  - High-quality technical blogs
  - Open-source documentation
  - Tutorial from recognized experts
  - Case studies from practitioners

- **Validation Criteria:**
  - ‚úÖ Author has proven track record
  - ‚úÖ Content matches academic consensus
  - ‚úÖ Practical examples can be verified
  - ‚úÖ No contradictions with Tier 1/2 sources

### Data Quality Checks

**Before Using Any Source:**
1. **Author Credentials Check:**
   - Search author name + domain
   - Verify academic credentials
   - Check publication history
   - Verify institutional affiliation

2. **Citation Analysis:**
   - Check Google Scholar citations
   - Review citing papers' quality
   - Look for critical responses
   - Verify claims haven't been refuted

3. **Cross-Reference Validation:**
   - Compare against 3+ other sources
   - Check for consensus vs. outlier
   - Verify experimental reproducibility
   - Look for peer commentary

4. **Recency Check:**
   - Prefer sources <5 years old
   - Check if superseded by newer research
   - Verify current relevance
   - Note field evolution speed

5. **Bias Detection:**
   - Check funding sources
   - Look for conflicts of interest
   - Identify commercial motivations
   - Verify through independent sources

## üéØ IMPLEMENTATION PROTOCOL

### Step 1: Domain Identification (5 minutes)

**Questions to Answer:**
- What is the core domain of this project?
- What adjacent domains are relevant?
- What level of expertise is required?
- What are the critical success factors?

**Example:**
```
Project: Build financial forecasting system for PE firm
Core Domain: Financial modeling, time series analysis
Adjacent Domains: Private equity, corporate finance, statistics
Expertise Level: Graduate-level quantitative finance
Critical Factors: Accuracy, interpretability, compliance
```

### Step 2: Curriculum Design (10 minutes)

**Create Learning Plan:**
1. List 5-10 foundational courses needed
2. Identify 20-30 key research papers
3. Find 3-5 authoritative textbooks
4. Select practical case studies
5. Identify validation sources

**Example Curriculum:**
```
UNDERGRADUATE:
- MIT 15.401: Finance Theory I
- Stanford CS229: Machine Learning
- Harvard STAT 110: Probability

GRADUATE:
- MIT 15.433: Investments
- Stanford MS&E 448: Big Financial Data
- Research: "Deep Learning for Financial Forecasting" (2024)

VALIDATION:
- CFA Institute standards
- SEC regulatory guidelines
- Industry benchmark comparisons
```

### Step 3: Rapid Learning Execution (30-60 minutes)

**For Each Learning Source:**

#### A. University Courses
```
1. Review course syllabus
2. Read lecture notes (all)
3. Study problem sets and solutions
4. Review exams and answers
5. Extract key frameworks and formulas
6. Note common mistakes and edge cases
```

#### B. Research Papers
```
1. Read abstract and conclusions
2. Analyze methodology section
3. Review results and limitations
4. Check citations and references
5. Look for replication studies
6. Note practical applications
```

#### C. Textbooks
```
1. Read table of contents
2. Study key chapters (not all)
3. Focus on frameworks and models
4. Review worked examples
5. Note best practices
6. Understand common pitfalls
```

### Step 4: Knowledge Synthesis (15 minutes)

**Create Mental Model:**
- Map relationships between concepts
- Identify core principles
- Recognize pattern applications
- Understand trade-offs and limitations
- Note when to use each approach

**Output: Expert Brief**
```markdown
## Domain Expert Brief: [Domain Name]

### Core Principles:
1. [Principle 1]
2. [Principle 2]
3. [Principle 3]

### Key Frameworks:
- Framework A: Used when [condition]
- Framework B: Used when [condition]

### Best Practices:
- Do: [practice 1]
- Don't: [anti-pattern 1]

### Common Mistakes:
1. [Mistake 1] ‚Üí Solution: [fix]
2. [Mistake 2] ‚Üí Solution: [fix]

### Validation Criteria:
- Output must meet: [standard 1]
- Acceptable range: [metric]
- Red flags: [warning signs]

### Sources Consulted:
- [20-50 validated sources listed]
```

### Step 5: Quality Validation (10 minutes)

**Self-Assessment:**
- [ ] Can I explain core concepts to an expert?
- [ ] Do I understand when NOT to use certain approaches?
- [ ] Have I identified controversial areas?
- [ ] Can I critique my own approach?
- [ ] Do I know current best practices?
- [ ] Have I learned from failures in the field?

**Validation Methods:**
- Cross-reference learning against 3+ authoritative sources
- Check understanding against recent expert debates
- Verify against industry standards
- Compare to current best practices

## About Wispr Flow

### What It Is
Wispr Flow is an AI-powered voice dictation and transcription app that:
- Transcribes speech to text in real-time across all apps
- **Flow Notes**: Captures spontaneous voice memos
- Syncs automatically across iPhone, Mac, and Windows
- Auto-adjusts tone based on app context
- Supports 100+ languages

### Key Features
- **Quick Capture**: Shortcuts to save ideas instantly
- **Cross-Device Sync**: Notes sync across all devices
- **App Integration**: Works inside Slack, Email, Docs, ChatGPT, etc.
- **Structured Output**: Transforms voice into formatted text

### Current Limitations
- ‚ùå No public API documented

- ‚ùå Manual export only (copy-paste)
- ‚úÖ Cross-device sync (but within Wispr ecosystem only)

## Proposed Solution

### Approach: Hybrid Automation System

Since Wispr Flow has no public API, we need a multi-pronged approach:

#### **Option 1: File System Monitoring (macOS/Windows)**
Monitor Wispr Flow's local storage for new notes and auto-export

**How it works:**
1. Detect Wispr Flow's data directory
2. Watch for new note files (likely SQLite or JSON)
3. Parse and extract note content
4. Push to integration endpoints (Notion, Airtable, etc.)

**Pros:**
- No API needed
- Real-time detection
- Works offline

**Cons:**
- Fragile (depends on app file structure)
- May break with app updates
- Privacy/security concerns

---

#### **Option 2: Clipboard Monitoring + Shortcuts**
Automate via system clipboard and keyboard shortcuts

**How it works:**
1. Create Wispr Flow shortcut to copy note to clipboard
2. Monitor clipboard for Wispr Flow note pattern
3. Auto-process and route to target systems
4. Use AppleScript/PowerShell for automation

**Pros:**
- Less invasive
- Doesn't depend on app internals
- Easy to maintain

**Cons:**
- Requires keyboard shortcut trigger
- Not fully automated (semi-manual)

---

#### **Option 3: Screen Scraping + OCR (Last Resort)**
Extract notes via UI automation

**How it works:**
1. Use Accessibility APIs (macOS) or UI Automation (Windows)
2. Open Wispr Flow app
3. Navigate to notes section
4. Extract text via accessibility tree or OCR
5. Parse and export

**Pros:**
- Works regardless of storage format
- Doesn't require file system access

**Cons:**
- Slow and brittle
- High maintenance
- Requires app to be open

---

#### **Option 4: Request Official API (Best Long-Term)**
Contact Wispr Flow team for API access

**Approach:**
1. Reach out to support@wisprflow.ai
2. Request API for enterprise/power users
3. Explain use case (AI consulting, automation)
4. Offer to beta test

**Pros:**
- Proper, supported integration
- Stable and maintained
- Could unlock premium features

**Cons:**
- May not exist yet
- Could be paid tier only
- Waiting on vendor

# Gmail API Setup Guide

Complete step-by-step instructions for setting up Gmail API credentials.

## Overview

You need to create OAuth 2.0 credentials to allow the Python script to access your Gmail account (read-only).

## Step-by-Step Instructions

### Step 1: Access Google Cloud Console

1. Go to: https://console.cloud.google.com/
2. Sign in with your Google account (mike.finneran@gmail.com)

### Step 2: Create a New Project

1. Click the **project dropdown** at the top of the page (next to "Google Cloud")
2. Click **"New Project"**
3. Enter project details:
   - **Project name**: `AI Research Automation` (or any name you prefer)
   - **Organization**: Leave as default
   - **Location**: Leave as default
4. Click **"Create"**
5. Wait for the project to be created (takes ~30 seconds)
6. Select the new project from the dropdown

### Step 3: Enable Gmail API

1. In the left sidebar, click **"APIs & Services"** > **"Library"**
   - Or use direct link: https://console.cloud.google.com/apis/library
2. In the search bar, type: `Gmail API`
3. Click on **"Gmail API"** from the results
4. Click the blue **"Enable"** button
5. Wait for it to enable (~10 seconds)

### Step 4: Configure OAuth Consent Screen

Before creating credentials, you need to configure the consent screen:

1. Go to **"APIs & Services"** > **"OAuth consent screen"**
   - Or direct link: https://console.cloud.google.com/apis/credentials/consent
2. Select **"External"** as User Type (unless you have Google Workspace)
3. Click **"Create"**
4. Fill in the required fields:
   - **App name**: `AI Research Report Fetcher`
   - **User support email**: Your email (mike.finneran@gmail.com)
   - **Developer contact email**: Your email
5. Click **"Save and Continue"**
6. On "Scopes" page, click **"Save and Continue"** (no changes needed)
7. On "Test users" page:
   - Click **"Add Users"**
   - Enter your email: `mike.finneran@gmail.com`
   - Click **"Add"**
   - Click **"Save and Continue"**
8. Review and click **"Back to Dashboard"**

### Step 5: Create OAuth 2.0 Credentials

1. Go to **"APIs & Services"** > **"Credentials"**
   - Or direct link: https://console.cloud.google.com/apis/credentials
2. Click **"+ Create Credentials"** at the top
3. Select **"OAuth client ID"**
4. If prompted to configure consent screen, follow Step 4 above first
5. Configure the OAuth client:
   - **Application type**: Select **"Desktop app"**
   - **Name**: `Gmail Report Fetcher Client` (or any name)
6. Click **"Create"**
7. A dialog will appear with your credentials - click **"Download JSON"**
8. Save the downloaded file

### Step 6: Save Credentials File

1. The downloaded file will have a name like:
   ```
   client_secret_123456789-abc123.apps.googleusercontent.com.json
   ```
2. Rename and move it to your knowledge base:
   ```bash
   mv ~/Downloads/client_secret_*.json ~/ai_research_knowledge_base/config/credentials.json
   ```

   Or manually:
   - Open Finder
   - Navigate to Downloads folder
   - Rename the file to `credentials.json`
   - Move it to: `/Users/mikefinneran/ai_research_knowledge_base/config/`

### Step 7: Verify File Location

Check that the file is in the right place:
```bash
ls -la ~/ai_research_knowledge_base/config/credentials.json
```

You should see the file listed.

### Step 8: First Authentication Run

Now run the script for the first time:

```bash
cd ~/ai_research_knowledge_base
python3 scripts/gmail_fetcher.py --days 7 --dry-run
```

This will:
1. Open your browser automatically
2. Ask you to sign in to Google
3. Show a warning that the app isn't verified (this is normal for personal projects)
4. Click **"Advanced"** > **"Go to [app name] (unsafe)"**
5. Click **"Allow"** to grant read-only Gmail access
6. Return to terminal - authentication complete!

The script will create `config/token.pickle` to remember your authentication.

### Step 9: Run for Real

Once authentication works, run without dry-run:
```bash
python3 scripts/gmail_fetcher.py --days 7
```

This will fetch and save any reports from the last 7 days.

## Troubleshooting

### "Access blocked: This app's request is invalid"
- Make sure you added yourself as a test user in OAuth consent screen
- Check that you selected "Desktop app" as application type

### "The file credentials.json is missing"
- Verify the file is in the correct location: `~/ai_research_knowledge_base/config/credentials.json`
- Check the file isn't empty and contains valid JSON

### "insufficient_scope" error
- Delete `config/token.pickle`
- Run the script again to re-authenticate with correct scopes

### Browser doesn't open automatically

- Copy and paste it into your browser manually
- Complete the authentication flow
- Copy the authorization code back to the terminal if prompted
